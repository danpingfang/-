个人对原型对象和原型链理解的总结

在理解原型对象和原型链之前我们需要先理解一下值传递和引用传递
在理解值传递和引用传递之前，首先我们得知道js的数据类型有数字、字符串、数组、对象、布尔、Null、Undefined
请看下面例子：

var a = 1;
var b = 2;
a = b;
b = 3;
console.log("a=",a,";","b=",b); //a= 2 ; b= 3
//说明数字是“值传递”的形式

var a = "111";
var b = "222";
a = b;
b = "333";
console.log("a=",a,";","b=",b); //a= 222 ; b= 333
//说明字符串是“值传递”的形式

var a = new Array();
a[0] = 1;
var b = new Array();
b[0] = 2;
a = b;
b[0] = 3;
console.log("a[0]=",a[0],";","b[0]=",b[0]); // a[0]= 3 ; b[0]= 3
//说明数组是“引用传递”的形式

var a = {};
a.name = "tom";
var b = {};
b.name = "lucy";
a = b;
b.name = "jack";
console.log("a=",a.name,";","b=",b.name); // a= jack ; b= jack
//说明对象是“引用传递”的形式

接着我们来看原对象的例子，如下

function Person () {
this.name = 'John';
} //这是一个构造函数，这里比喻成大楼图纸
var person = new Person(); //构造实例对象，这里可以比喻成根据大楼图纸建造的一幢大楼
Person.prototype.say = function() {
console.log('Hello,' + this.name);
};
 person.say();//Hello,John

Person.prototype就是指对象原型，
Person.prototype.say就是给对象原型添加了一个say的方法
对于紫色部分代码这里比喻成通过在原来的图纸添加一些新的设计（方法／属性）来对原来的大厦进行优化
因此大厦只是添加了一些新功能,原来功能还是在的

function Person () {
this.name = 'John';
} //这是一个构造函数，这里比喻成大楼图纸
var person = new Person();//构造实例对象，这里可以比喻成根据大楼图纸建造的一幢大楼
Person.prototype = {
say: function() {
console.log('Hello,' + this.name);
}
};
 person.say();//person.say is not a function

对于紫色部分代码可以理解成重新绘制了一张图纸来建造新的大楼
因此原来的大楼是找不到say这个功能的
